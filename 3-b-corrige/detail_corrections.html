Exercice 1 : Structure de la page

Structure HTML5 pertinente

  • Un élément <header> est ajouté autour de la zone d'en-tête.
  • Des éléments <nav> sont ajoutés autour du ou des lien(s) d'accès rapide et autour du menu de navigation.
  • Un élément <main> est ajouté autour de la zone de contenu principal.
  • Optionnellement, un élément <aside> est ajouté autour de la zone « Voir aussi » (cet élément aurait pu être inclu directement dans l'élément <main>).
  • Un élément <footer> est ajouté autour de la zone de pied de page.

Éléments de navigation interne

Landmarks

  • Un role="banner" est défini sur l'élément <header>.
  • Un role="search" est défini sur la <div> ou sur le <form> contenant le champ de recherche.
  • Des role="navigation" sont définis sur les éléments <nav>.
    Pas obligatoire mais recommandé : des propriétés aria-label sont également définies sur ces éléments afin de les nommer. Par exemple : aria-label="Accès rapide" et aria-label="Menu principal".
  • Un role="main" est défini sur l'élément <main>.
  • Un role="complementary" est défini sur l'élément <aside> s'il existe.
  • Un role="contentinfo" est défini sur l'élément <footer>.

Liens d'accès rapide

  • Au moins un lien (ancre) est ajouté au début de la page dans le code HTML et mène vers le contenu principal.

Exercice 2 : Contenu

Éléments de contenu

Titres

  • Un <h3> est ajouté sur les textes : « Pas de ROI sur l'optimisation » et « Des vitesses d'accès inchangées ».
  • Un <h2> est ajouté sur le texte « Votez pour le logiciel le plus gourmand ! ».
  • Un <h1> est ajouté sur le texte « La loi de Moore » dans la modale.

Listes

  • Une liste <ul>/<li> est utilisée pour structurer les liens du menu de navigation.
  • Une liste <ul>/<li> est utilisée pour structurer les liens de la section « Voir aussi ».
  • Une liste <ul>/<li> est utilisée pour structurer les liens des réseaux sociaux en pied de page.

Images

  • L'attribut alt="vue macro d'un circuit intégré" est vidé pour laisser un alt="" uniquement, l'image étant purement décorative, elle doit être ignorée par les technologies d'assistance.
  • La structure <figure>/<figcaption> avec ses propriétés ARIA associées est utilisée autour de l'image loi-moore.gif et sa légende dans la fenêtre modale. En effet, cette image est une image légendée car le texte adjacent apporte des informations sur l'image (copyright).

Tableau

  • Un texte « Résultats des votes » est ajouté directement après la balise <table> ouvrante, dans un élément <caption>.
  • Des éléments <th> remplacent les <td> de la première ligne et optionnellement de la première colonne du tableau.
    Si des éléments <th> sont utilisés dans la première colonne, alors des attributs scope sont ajoutés sur tous les éléments <th> du tableau (avec pour valeurs : col dans la première ligne, row dans la première colonne).

Liens

  • Ajout d'un title sur chacun des deux liens « En savoir plus ». En effet, ces liens partagent le même contexte. Ce sont donc des liens identiques qui mènent vers des pages différentes. Il est nécessaire de les différencier, par exemple en ajoutant un attribut title qui reprend le nom visible avec une information complémentaire : « En savoir plus sur le syndrome de l'obésiciel » et « En savoir plus sur la lenteur ressentie par l'utilisateur ».
  • En pied de page, l'intitulé des liens des réseaux sociaux est donné en donnant une alternative aux images svg. Pour une correction complète :
    • ajout du role="img" sur chaque balise <svg> ;
    • ajout d'une propriété aria-label sur chaque balise <svg>  : compter twitter, compte linkedin, envoyer un mail etc.

Information par la couleur

Le lien de la page en cours de consultation est identifié dans le menu de navigation (« Loi de Wirth »).

Il n'y a pas de problèmes pour les personnes malvoyantes, puisque l'inversion des couleurs par rapport aux liens inactifs (comme « Loi de Murphy » par exemple), équivaut à un changement de forme. Une personne malvoyante pourra donc percevoir la différence entre le lien actif et les autres.

On ajoute par contre un title sur le lien pour ajouter l'information pour les personnes aveugles naviguant avec un lecteur d'écran. Attention, le title doit toujours reprendre le nom accessible du lien : title="Loi de Wirth (page active)"

Exercice 3 : Formulaires

Regroupements

  • Le texte « Quels sont les logiciels les plus gourmands selon-vous ? » et les cases à cocher sont regroupés dans un élément <fieldset>.
  • Un élément <legend> remplace le <p> autour du texte « Quels sont les logiciels les plus gourmands selon-vous ? ».
  • Une correction similaire est appliquée pour « Quels sont les logiciels les moins gourmands selon-vous ? ».

Étiquettes

  • Au niveau du moteur de recherche : ajout d'un attribut for="search" sur la balise <label> pour donner un nom accessible au champ de saisie.
  • Pour tous les champs (texte, zone de texte et case à cocher) du formulaire « Votez pour le logiciel le plus gourmand ! », des étiquettes sont définies au moyen d'éléments <label for="id"> ;

Identifications de données personnelles

  • Des attributs autocomplete sont ajoutés sur les champs qui collectent des données personnelles :
    • Nom : autocomplete="family-name"
    • Prénom : autocomplete="given-name"
    • Adresse e-mail : autocomplete="email"

Intitulé des boutons

  • La propriété aria-label du bouton « Voter maintenant ! », est supprimée. Son contenu (« Envoyer ») ne reprenait pas au moins le nom visible (« Voter maintenant ! »).

Aides à la saisie

Pour le formulaire « Votez pour le logiciel le plus gourmand ! » :

  • Champs obligatoires :
    • Un astérisque est ajouté dans l'étiquette du champ « Nom » qui est obligatoire mais qui n'est pas identifié visuellement (uniquement par aria-required) ;
    • Un texte visible « * champs obligatoires » est ajouté avant le début du formulaire.
  • Le format de saisie présent dans la page « format : nom@domaine.com » et relié au champ « adresse email » par une propriété aria-describedby sur le champ.
  • Le message d'erreur du champ email est modifié pour afficher un exemple réel de saisie « Saisissez un email valide, par exemple jean@dupont.fr » (le code JavaScript qui gère cet élément est dans le fichier js/form.js).

Exercice 4 : Javascript et ARIA

Mise à jour dynamique de contenu

Au niveau du moteur de recherche : les propriétés aria-live="polite" et aria-atomic="true"sont ajoutées directement dans la page HTML, sur l'élément qui contient les résultats de recherche (#results-search-total).

Disclosure

Bouton « Voir les derniers résultats » :

Les modifications ont été faites dans le fichier js/disclosure.js. Voir le script commenté.

Les propriétés aria-expanded et optionnellement aria-controls sont ajoutées et le code permet de modifier les valeurs dynamiquement.

Attention, garantir la présence de aria-expanded avec la valeur false dès le chargement de la page (en l'absence d'interaction de l'utilisateur) pour garantir l'accès à l'état replié dès le début

Fenêtre modale

Le bouton « La loi de Moore en 2 secondes » gère une fenêtre modale.

Les propriétés suivantes sont ajoutées sur l'élément #modal-content directement dans le HTML :

  • role="dialog"
  • aria-labelledby="modal-title"

Les autres modifications ont été faites dans le fichier js/dialog.js. Voir le script commenté.

  • À l'ouverture de la modale, on force la reprise de focus sur le bouton de fermeture de la fenêtre modale : btnClose.focus(); dans la fonction btnOpen.addEventListener
  • À la fermeture de la modale, on force la reprise de focus sur le bouton d'ouverture dans la page « La loi de Moore en 2 secondes ») : btnOpen.focus(); dans la fonction endFunction().

Contrôle de saisie de formulaire

Les modifications ont été faites dans le fichier js/form.js. Voir le script commenté.

On ajoute la propriété aria-describedby aux champs qui affichent des erreurs afin de lier les messages d'erreurs aux champs : obj.setAttribute( 'aria-describedby', label );.

Envoi du formulaire avec succès

  • Après la soumission du formulaire complet, sans erreurs, la méthode focus() est utilisée pour déplacer le focus de l'utilisateur sur le texte de succès ajouté également par le script : success.focus();.
  • Pour que la reprise de focus soit fonctionnelle, on ajoute une propriété tabindex="-1" au conteneur visé par la reprise de focus.
