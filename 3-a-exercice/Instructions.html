Exercice 1 : Structure de la page

Structure HTML5 pertinente

Vous devez implémenter une structure HTML5 pertinente à l’aide des balises de structure HTML5.

Note : les éléments <section> , <article> ou encore <aside> ne compteront pas dans l’évaluation de cet exercice, seuls les éléments obligatoires au regard du RGAA seront pris en compte.

Navigation interne

Vous devez implémenter les éléments de navigation interne :

  • Au moins un lien d’accès rapide au contenu ;
  • Tous les landmark ARIA nécessaires et obligatoires dans le RGAA.

Exercice 2 : Contenu

Éléments de contenu

Réparez les éléments de contenu suivants.

  • La hiérarchie des titres : vérifiez que la hiérarchie des titres est cohérente et que tous les éléments structurants de la page sont correctement identifiés ;
  • Les listes d'éléments : vérifiez que les éléments qui se présentent sous forme de liste sont correctement implémentés ;
  • Les images :
    • vérifiez que les images de décoration sont correctement implémentées pour être ignorées par les technologies d'assistance ;
    • vérifiez que les images porteuses d'informations sont correctement identifiées et que leurs alternatives sont pertinentes.
  • Les tableaux : vérifiez que les tableaux sont correctement structurés avec des titres et des entêtes notamment ;
  • Les liens.

Information par la couleur

Une information est véhiculée par la couleur dans la page. Repérez-la et mettez en place une méthode de votre choix pour que tous les utilisateurs aient accès à cette information.

Exercice 3 : Formulaires

Il y a deux formulaires dans la page, la recherche et « Votez pour le logiciel le plus gourmand ! », vous devez contrôler et réparer, si nécessaire :

  • La structure des formulaire ;
  • Les étiquettes de tous les champs de formulaire ;
  • Les identifications de données personnelles ;
  • Les intitulés de boutons ;
  • Les aides à la saisie (champs obligatoires, formats et exemples réels).

Exercice 4 : Javascript et ARIA

De manière générale pour les participants qui ne pratiquent pas JavaScript : le jour de l'examen de certification, vous pourrez décrire une correction précise pour chacun des exercices de cette section. Une zone de texte dédiée sera disponible. La correction que vous fournirez devra permettre à une personne maîtrisant JavaScript de réparer parfaitement les éléments.

Mise à jour dynamique de contenu

Le bouton « Rechercher » affiche un nombre de résultats trouvés ainsi qu’une liste de résultats. Au regard des obligations du RGAA concernant les mises à jour dynamiques de contenu, implémentez une méthode de votre choix pour vous assurer que les utilisateurs qui en ont besoin seront prévenus de cette mise à jour et que le message complet sera toujours restitué.

Concernant les mises à jour dynamiques de contenu, attention à vérifier la compatibilité de vos implémentations avec les lecteurs d'écrans. Une série de tests réalisés par Access42 peut vous aider à choisir l'implémentation la plus robuste.

Disclosure

Le bouton « Voir les derniers résultats » gère l’affichage du tableau des résultats du vote.

Vous devez rendre accessible ce composant.

Le code JavaScript qui gère cet élément se trouve dans le fichier js/disclosure.js.

Le motif de conception correspondant est consultable à cette adresse : Disclosure (nouvelle fenêtre).

Fenêtre modale

Le bouton « La loi de Moore en 2 secondes » gère une fenêtre modale.

Vous devez vérifier que le motif de conception de la fenêtre modale est correct et réparer les erreurs d’implémentation HTML, mais également JavaScript.

Le code JavaScript qui gère cet élément est dans le fichier js/dialog.js.

Le motif de conception de la fenêtre modale est consultable à cette adresse : Dialog modal (nouvelle fenêtre).

Contrôle de saisie de formulaire

Lorsque le formulaire est soumis et contient des erreurs, le contrôle de saisie affiche des messages d’erreur sur les champs « Nom » et « Adresse email ».

Vous devez implémenter une méthode de votre choix pour garantir que ces messages d’erreurs seront correctement restitués à la prise de focus sur les champs en erreur.

Le code JavaScript qui gère cet élément est dans le fichier js/form.js.

Envoi du formulaire avec succès

Lorsque vous soumettez le formulaire correctement, le formulaire disparaît et le message de succès apparaît.

Vous devez vous assurer que l’utilisateur aura conscience de l’apparition de ce message et qu’il sera repositionné au bon endroit dans la page.

Le code JavaScript qui contrôle l’apparition du message est dans le fichier js/form.js.

Exercice 5 : Validité du code

Une fois votre page réparée, vous devez vous assurer d’avoir un code source généré qui soit valide.
